Raising Awareness of SQL Injection and Protecting Data

by Mark McQuillen

August 10, 2014

What is SQL Injection?

SQL injection is a method of hacking the databases of organizations through the fields on their websites that are meant to being populated by the public. Hackers enter SQL commands into forms and then gain access to the database and are able to read and even delete tables.

Why is that bad?

This is a major problem because hackers can access information that is meant to be private. This could include credit card numbers or any sensitive information. If a hacker deletes tables from a database that can cause untold harm to a company when those records are lost.

How Do New Programmers Prevent SQL Injection?

There are a number of ways beginning programmers can protect against SQL Injection. Here are two:

1. Prepared Statements (Parameterized Queries)
This means that a field will only accept a pre-determined input statements. If a user tries to enter a statement that is not approved, it will not be accepted.

2. Escaping All User Supplied Input
This means that user input does not have access to the database. It is first rewritten into sanitized prepared statements that are safe. Unfortunately, this approach can have a negative effect on performance and can be troublesome to retrofit.
