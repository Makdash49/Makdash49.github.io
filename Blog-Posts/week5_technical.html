A Thing or Two About Classes

by Mark McQuillen

June 15, 2014

Classes of Spaceships

Classes are a way of defining objects so that you can create instances of a class and then call those instances with methods defined as part of the class.

Here's an example:

We could define a class called: spaceships
This class would take arguments as parameters such as: purpose, speed, crew-size. Then you would want to initialize these variables so that they can be used in all the methods of the class. You do this by using the "@" symbol which means this is an instance variable. You initialize a method like this:

def initialize (purpose, speed, crew)
@purpose = purpose
@speed = speed
@crew = crew

To later create instances of class you would name it like this while passing in arguments:

x-wing = spaceships.new(fighter, hyperspace, 1)
space_shuttle = spaceships.new(science, mach 10, 8)
borg_cube = spaceships.new(domination, Warp 9, 100,000)

Now that we have some instances it would be nice to be able to call some methods on them, for instance:

x-wing.function
borg_cube.population
space_shuttle.kessel_run
Back in the body of the class spaceship you define these methods.

For instance:

def population
if crew < 2
puts "It's lonely in space. No one can hear you scream, or sneeze, or even yawn for that matter."
elsif crew < 50
puts "Sounds like a nice Sunday Drive."
else
puts "What is this? The Love Boat?!"
end
end

So this is what would happen:

x-wing.population => "It's lonely in space. No one can hear you scream, or sneeze, or even yawn for that matter."

borg_cube.population => "What is this? The Love Boat?!"

space_shuttle.population => "Sounds like a nice Sunday Drive."

* I should mentioned that Ruby has some built in classes like Array which you can modify in a program.
