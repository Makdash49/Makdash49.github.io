Comparing Object Oriented Programming and Functional Programming

by Mark McQuillen

July 31, 2014

What is Object Oriented Programming?

Object-Oriented Programming is an approach to coding that uses levels of classes with defined objects that work together. Classes define methods that work on data.

What is Functional Programming?

Functional Programming is an approach to coding that defines functions rather than on performing operations or defining objects.

How do you decide if you are going to use Object-Oriented Programming or Functional Programming?

You decide depending on how you foresee the evolution of your software. If you believe your code will have a set number of operations but you will add new things, then it would be good to use Object-Oriented Programming. Existing classes are left alone while new classes are added that use existing methods.

In contrast, Functional Languages are preferred if, as your code evolves, you have a set number of things but you add new operations to those existing things. This is done by adding new functions while leaving the existing functions alone.
